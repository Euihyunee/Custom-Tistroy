# Project Rules

이 프로젝트와 관련된 모든 작업에 대해 에이전트는 다음 규칙을 준수해야 합니다.

## 1. 언어 및 커뮤니케이션
- **모든 출력은 한국어(Korean)로 작성합니다.** (코드 내 주석 포함 권장)
- 기술적 용어는 통용되는 영문을 병기하거나 그대로 사용할 수 있습니다 (예: SEO, Container, Dependency Injection).

## 2. 프로젝트 목표 (티스토리 블로그 커스터마이징)
이 프로젝트는 `Euihyunee/Custom-Tistroy` 스킨을 기반으로 다음 세 가지 핵심 가치를 극대화하는 것을 목표로 합니다.

### A. SEO 최적화 (검색 엔진 친화적)
- 검색 엔진이 콘텐츠를 효과적으로 인덱싱할 수 있도록 구조화된 데이터(JSON-LD)를 적극 활용합니다.
- 메타 태그(Open Graph, Twitter Cards)를 최적화합니다.

### B. AI 가독성 강화 (LLM Friendly)
- AI 에이전트가 정보를 정확히 추출할 수 있도록 시멘틱 마크업(`Semantic HTML`)을 엄격히 준수합니다.
- 필요 시, AI 전용 요약 블록(Hidden Summary)을 게시글에 포함합니다.

### C. 개발자 친화적 기능 (Dev Blog)
- 코드 블록의 가독성과 기능(복사, 라인 넘버, 언어 표시)을 최우선으로 합니다.
- 수식(LaTeX), 다이어그램(Mermaid) 등 기술 블로그에 필요한 도구를 통합합니다.

## 3. 코드 작성 가이드라인
- **PugJS**: HTML 구조는 Pug 템플릿을 사용하여 간결하게 작성합니다.
- **TailwindCSS**: 스타일링은 유틸리티 클래스 위주로 작성하되, 복잡한 컴포넌트는 `@layer components` 등을 활용해 추상화합니다.
- **AlpineJS**: 프론트엔드 인터랙션은 가벼운 AlpineJS를 사용하여 구현합니다.
